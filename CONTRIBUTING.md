# Contributing

## Branches

- Use `codex/<short-description>` for automation branches.
- Keep `main` releasable.

## Commits

- Follow Conventional Commits.
- Run `pnpm lint`, `pnpm typecheck`, and `pnpm test` before opening a PR.

## Pull Requests

1. Create a focused branch.
2. Keep changes small enough to review.
3. Include test evidence and any deployment notes.
4. Do not merge with failing CI.
